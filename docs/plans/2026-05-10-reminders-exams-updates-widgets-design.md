# 提醒、考试、更新与小组件刷新设计

## 目标

补齐临时取消单节课、灵活时段首课提醒、考试类别与考试提醒、GitHub Release 更新检查/下载，以及小组件午夜切日和 5 分钟静默刷新能力。

## 临时取消单节课

现有 `TemporaryScheduleOverride` 只表达“某天按另一日期上课”。新增正式规则类型字段，不把取消逻辑做成 UI 层过滤。规则支持：

- 调课：目标日期显示来源日期课程，兼容既有字段。
- 取消：目标日期取消指定节次范围，可选限定课程 ID。

课表页、小组件数据源、提醒规划器都通过同一个内核判断函数过滤课程。这样同一条取消规则会同时影响显示、今日/下一节小组件和闹钟计划。

## 灵活时段首课提醒

> 已被 `docs/plans/2026-05-10-flexible-first-course-reminder-design.md` 中的“条件规则 + 自定义时间段占用”方案取代。本节仅保留为历史设计记录。

保留 `FirstCourseOfPeriod` 的提醒类型，扩展规则配置：

- 每个时段有节次范围，例如上午、下午、晚上。
- 每个时段可配置一组“免提醒前置节次”。
- 如果当天前置节次里有课，该时段不再产生首课提醒。
- 如果当天前置节次没有课，则提醒该时段后续第一门课。

这覆盖“有早自习则早自习不提醒且当天上午不再提醒；没有早自习则上午第一节提醒；有早自习配置但当天早自习为空，则后续上午第一门课提醒”的情况。UI 不使用“从第几节开始算首课”这种表述，而使用“免提醒前置节次”和“时段范围”。

## 考试类别与考试提醒

`CourseItem` 增加课程类别，默认是普通课程，新增考试类别。手动添加课程可选择类别；插件同步数据如果未给类别则保持普通课程。

考试卡片在课表中使用更醒目的边框、颜色和“考试”标记。课程详情、单课提醒、批量提醒会根据类别调整文案；考试默认提前 40 分钟，普通课程保持默认提前 20 分钟。

## 检查更新与自动更新

发布流程新增 `update.json` 资产，与 APK 一起发布到 GitHub Release。清单包含：

- `versionCode`
- `versionName`
- `tagName`
- APK 文件名、ABI、sha256、下载 URL

App 检查更新流程：

1. 请求 GitHub `releases/latest`。
2. 如果没有 Release 或没有 `update.json`，显示“暂无发布版本”。
3. 下载并解析 `update.json`。
4. 只有远端 `versionCode > BuildConfig.VERSION_CODE` 才判定有更新。
5. 按设备 ABI 选择最合适 APK，源站和镜像站只参与测速/下载通道选择，不参与版本判断。

目前仓库没有任何 Release，因此实现后的正确表现是“不误报更新，明确提示暂无发布版本”。

候选源包含源站、用户提供的 GitHub 镜像站，以及 jsDelivr CDN。运行时测速成功的源才会参与下载；`v6.gh-proxy.org` 按用户实测纳入候选。

自动更新是可选项，表现为自动检查、自动下载可用 APK，并调用 Android 标准安装确认界面。普通第三方应用不能静默安装 APK。

## 小组件刷新

小组件夜间自动切到下一天的阈值从 22:00 改为 00:00，避免晚上无法往回翻。

新增 5 分钟刷新调度，使用显式 `AlarmManager` 广播驱动；保留 WorkManager 作为系统允许的周期健康调度。刷新时先重新读取时间、课程、考试和提醒数据，再更新小组件。刷新本身不展示提示，不弹 UI；数据未变化时生成的内容保持稳定，避免用户肉眼看到闪动。

## GitHub Action

Release workflow 构建并重命名 APK 后计算 sha256，生成 `update.json`，再把 `ClassScheduleViewer-*.apk` 和 `update.json` 一起上传 artifact 并发布到 GitHub Release。

## 测试

- 内核测试：取消规则过滤、时段首课提醒前置节次、考试提醒默认值。
- 小组件测试：00:00 前后自动切日逻辑、取消课/考试数据进入小组件。
- 更新测试：无 Release、无 `update.json`、版本相同、版本更高、ABI 匹配、镜像测速排序。
- 构建测试：相关单测与 `assembleDebug`。
