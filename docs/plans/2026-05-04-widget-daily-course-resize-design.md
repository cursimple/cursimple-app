# 桌面小组件每日课程与尺寸适配设计

日期：2026-05-04  
状态：已确认  
范围：每日课程、下一节课、课程提醒三个桌面小组件

## 1. 目标

- 将“今日课程”桌面小组件统一改名为“每日课程”。
- 每日课程支持连续切换任意日期，不再限制为昨天、今天、明天。
- 三个桌面小组件都支持用户在桌面上拖拽调节大小。
- 三个桌面小组件至少面向 2x2 和 3x3 尺寸做内容适配。
- 每日课程、下一节课、课程提醒的条目样式互相统一，去掉左侧偏蓝色强调条。

## 2. 推荐方案

采用单 provider + 系统可调尺寸 + 内容分档适配。

每个小组件保留一个正式入口，继续通过 `appwidget-provider` 声明 `resizeMode="horizontal|vertical"`、`minResizeWidth`、`minResizeHeight`、`targetCellWidth`、`targetCellHeight`。Android 桌面负责长按拖拽调节尺寸，应用负责声明允许的尺寸边界并在不同尺寸下渲染合适内容。

相比为每种网格尺寸注册多个 provider，这个方案不会让系统小组件选择器膨胀出大量重复入口，也能保持现有厂商桌面适配链路稳定。

## 3. 设计边界

- 不把“临时兜底”作为正式架构手段。
- 不新增大量重复 provider 作为尺寸方案。
- 不改变课表、提醒、插件同步的数据模型。
- 不改变小组件刷新来源，仍复用现有 DataStore 快照与刷新广播。
- 不在文档写入后执行 git commit，提交由用户明确发起。

## 4. 每日课程切换

当前限制来自 `DataStoreWidgetPreferencesRepository` 中的偏移范围，以及 `ScheduleGlanceWidgetReceiver` 中左右按钮显隐逻辑。

改造后：

- 移除 `-1..1` 的硬限制，保留一个明确的极大安全边界防止整型异常。
- 上一天、下一天按钮始终显示并可点击。
- 非今天时显示“回今天”。
- 标题格式保持为 `M月d日 · 标签`：
  - `0` 显示“今天”
  - `1` 显示“明天”
  - `-1` 显示“昨天”
  - 正数显示 `+N天`
  - 负数显示 `N天`

## 5. 小组件尺寸适配

三个小组件都声明可横纵向调节尺寸。

默认目标尺寸：

- 每日课程：默认 3x3，最小覆盖 2x2。
- 下一节课：默认 2x2，允许拉大到 3x3 及以上。
- 课程提醒：默认 3x2，允许拉大到 3x3 及以上。

内容适配：

- 2x2：展示标题和最关键的 1 到 2 条内容，减少辅助文本。
- 3x3：展示更完整的列表，保留地点、教师、倒计时等辅助信息。
- 更大尺寸：增加可见条目数量，但不改变信息架构。

每日课程是 RemoteViews 实现，需要通过 `onAppWidgetOptionsChanged` 读取尺寸并刷新；下一节课和课程提醒是 Glance 实现，可通过 Glance 尺寸 API 做分档内容。

## 6. 样式统一

每日课程的课程行移除 `course_accent` 左侧竖条，改为完整圆角浅色内容块。

下一节课和课程提醒当前复用 `AccentRow`，该组件也去掉左侧强调条，改为统一的浅色条目容器。状态信息继续通过文字标签、颜色和徽章表达，不依赖左侧色条。

## 7. 验证方式

- 运行 `pwsh -NoLogo -NoProfile -Command './gradlew :feature-widget:assembleDebug :app:assembleDebug'`。
- 检查“今日课程”文案不再出现在小组件名称、选择页标题和描述里。
- 检查每日课程可从今天连续切换到至少前后多天，并可回到今天。
- 手动在桌面拖拽每日课程、下一节课、课程提醒，确认 2x2 和 3x3 下内容不重叠。
- 检查三个小组件条目左侧不再出现偏蓝色竖条。
