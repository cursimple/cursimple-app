# 课表卡片临时取消设计

## 背景

应用已经支持通过设置页创建 `TemporaryScheduleOverride.CancelCourse` 规则，并且小组件会过滤被临时取消的课程。当前课表页也会直接过滤这些课程，导致用户无法从课表卡片快速取消或恢复某一节课。

## 目标

- 短按有课的课表框后，在现有课程详情弹窗中显示“临时取消”按钮。
- 如果该课程在当前实际日期已经处于临时取消状态，则显示“取消临时取消”按钮。
- app 内课表保留已取消课程，并用对角线标记为已取消。
- 小组件继续不显示被临时取消的课程。
- 对应课程提醒自动失效，不再生成该次课程提醒。

## 方案

复用已有的 `TemporaryScheduleOverride.CancelCourse` 数据模型，不新增独立状态表。课表点击时携带课程对应的实际日期，详情弹窗按“日期 + 课程”查找匹配的取消规则；新增规则写入课程 ID、实际日期和课程节次，恢复时删除匹配规则。

app 课表渲染不再过滤已取消课程，而是在周视图和日视图中计算 `temporarilyCancelled` 标记并绘制对角线。小组件数据源保持现有 `filterTemporaryCancelledCourses` 行为，因此仍隐藏已取消课程。

提醒规划器继续使用已有 `isCourseTemporarilyCancelled` 判断；课表短按创建的规则写入课程 ID，因此只让该课程对应提醒失效。

## 验证

- 编译 `feature-schedule`，确认新增参数和 Compose UI 可通过编译。
- 运行相关单测，确认已有临时取消、小组件过滤逻辑不回退。
