# 自动检查更新与小组件提醒注册就绪设计

## 目标

把“自动更新”明确调整为“自动检查更新”：自动检查只发现并提示新版本，不自动下载、不自动安装。小组件每 5 分钟刷新时，继续按当前提醒后端设置同步提醒，并把系统提醒注册结果记录为清晰状态。

## 更新检查

设置页提供检查更新入口和自动检查更新开关。自动检查触发时，请求 GitHub Release 与 `update.json`，只有远端 `versionCode` 高于当前版本才提示用户。

发现新版本时展示：

- 新版本号。
- 变更内容，优先使用 `update.json` 中的变更字段，没有则使用 GitHub Release 正文。
- 操作按钮：更新、忽略本次更新。

用户点“更新”后才下载 APK，下载完成后打开 Android 系统安装确认界面。用户点“忽略本次更新”后，记录被忽略的 `versionCode`；后续自动检查不再提示这个版本，但手动检查仍可显示并更新。

## 小组件 5 分钟检查

现有提醒注册后端不重做。小组件 5 分钟检查继续调用现有 `ReminderCoordinator.syncAlarmsForWindow`，实际提交目标由当前提醒设置决定：

- 系统闹钟 App 后端：提交到系统闹钟 App。
- App 自管闹钟后端：提交到已有自管闹钟调度。

同步完成后根据结果记录明确状态：

- 已注册就绪：应注册的提醒已存在。
- 已补注册：本轮补注册成功。
- 注册失败：提交失败、不可表示或登记失败。
- 无需注册：当前窗口没有可注册提醒。

该状态用于日志和后续展示，不作为新的架构分支，也不引入临时兜底。

## 测试

- 更新模型和检查器测试：解析变更内容、手动检查无视忽略版本、自动检查跳过被忽略版本。
- 设置/关于页面编译检查：检查入口、弹窗与按钮状态。
- 小组件同步测试：已有提醒、补注册成功、失败时状态文案正确。
- 运行相关单测与 `assembleDebug`。
