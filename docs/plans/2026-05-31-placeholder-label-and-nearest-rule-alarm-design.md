# 占位课 Label 与新规则最近闹钟设计

## 目标

占位课创建时，Label 字段应能从已有 label 下拉选择，也允许直接输入新 label。新建提醒规则完成后，应立即注册该规则在未来 48 小时内最近的一次闹钟，而不是只检查当天窗口。

## 方案

占位课弹窗继续由 `PlaceholderCourseDialog` 负责。将原来的文本框加“切换已有 label”按钮改为可编辑的 Material3 下拉输入框，选项来自现有 `slotLabels`，输入内容仍作为最终保存值。

提醒注册逻辑放在提醒核心层。`ReminderCoordinator` 增加“同步最近一个规则闹钟”的方法：只展开指定规则，窗口为当前时刻到未来 48 小时，取最近的一条提醒计划并注册。`ScheduleViewModel.saveLabelReminderRule` 通过 `ruleId == null` 区分新建与编辑；新建走最近闹钟同步，编辑继续沿用现有当天重算逻辑。

## 边界

- 只针对 label 规则新建入口。
- 最近闹钟只来自刚保存的规则。
- 48 小时内没有计划时，不创建闹钟，沿用现有摘要消息。
- 系统闹钟后端仍遵守当前“系统时钟可表达日期”的限制。

## 测试

新增或扩展单元测试覆盖：新建规则只注册未来 48 小时内最近一条；超出 48 小时的提醒不会被立即注册；既有同步逻辑保持不变。
